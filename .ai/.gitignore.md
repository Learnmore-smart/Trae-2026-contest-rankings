# .gitignore

> Last updated: 2026-07-03 | Protection: STANDARD

## Purpose

Keep local development artifacts, generated caches, logs, screenshots, and secrets out of git while preserving source, tests, generated SDK code, and public assets that are intentionally versioned.

## What It Does

- Ignores dependency installs and Next.js build output.
- Ignores local environment files and service-account secrets.
- Ignores logs, PID files, dev-server job markers, Playwright MCP captures, test result screenshots, TypeScript build info, and local contest scrape caches.
- Keeps `.env.example`, checked-in public assets, application code, tests, and Data Connect generated client files trackable.

## Public API

| Name | Type | Description |
|------|------|-------------|
| `.gitignore` | Git config | Defines repository-level ignore rules for local/generated artifacts. |

## Props / Parameters

| Name | Type | Required | Description |
|------|------|----------|-------------|
| ignore patterns | glob lines | Yes | Patterns evaluated by git for untracked files. |

## Dependencies

- **Internal:** `package.json` - build/test scripts create artifacts covered by these rules.
- **Internal:** `lib/trae/topics-cache.json` - large local scrape cache that must not be versioned.

## Agent Decisions / Thoughts

- **2026-07-03 Codex:** Treat `dev-server*.pid`, `dev-server*.job`, `.playwright-mcp/`, `test-results/`, logs, and `lib/trae/topics-cache.json` as disposable local/runtime artifacts. Root screenshots and `public/` assets are left trackable because they may be documentation or UI assets.

## Important Notes / NEVER Change

- Do not ignore `.env.example`; it documents required environment variables.
- Do not ignore `lib/dataconnect-generated/`; the app depends on the checked-in generated package.
- Do not ignore `public/`; it contains intentional UI assets.
