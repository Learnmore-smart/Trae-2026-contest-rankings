# .gitignore

> Last updated: 2026-07-03 | Protection: STANDARD

## Purpose

Keep local development artifacts, generated caches, logs, screenshots, and secrets out of git while preserving source, tests, generated SDK code, compact fallback data, and public assets that are intentionally versioned.

## What It Does

- Ignores dependency installs and Next.js build output.
- Ignores local environment files and service-account secrets.
- Ignores logs, PID files, dev-server job markers, Playwright MCP captures, test result screenshots, and TypeScript build info.
- Keeps `.env.example`, checked-in public assets, application code, tests, Data Connect generated client files, and the compact `lib/trae/topics-cache.json` fallback snapshot trackable.

## Public API

| Name | Type | Description |
|------|------|-------------|
| `.gitignore` | Git config | Defines repository-level ignore rules for local/generated artifacts. |

## Props / Parameters

| Name | Type | Required | Description |
|------|------|----------|-------------|
| ignore patterns | glob lines | Yes | Patterns evaluated by git for untracked files. |

## Dependencies

- **Internal:** `package.json` - build/test scripts create artifacts covered by these rules.
- **Internal:** `lib/trae/topics-cache.json` - compact fallback snapshot that must remain versioned for offline/Data Connect-unavailable ranking behavior.

## Agent Decisions / Thoughts

- **2026-07-03 Codex:** Treat `dev-server*.pid`, `dev-server*.job`, `.playwright-mcp/`, `test-results/`, and logs as disposable local/runtime artifacts. Root screenshots and `public/` assets are left trackable because they may be documentation or UI assets.
- **2026-07-03 Codex:** Do not ignore `lib/trae/topics-cache.json`; tests and runtime fallback paths require a committed compact snapshot when Data Connect credentials are missing. Raw scrape exports should use a different local filename or directory covered by existing log/tooling ignores.

## Important Notes / NEVER Change

- Do not ignore `.env.example`; it documents required environment variables.
- Do not ignore `lib/dataconnect-generated/`; the app depends on the checked-in generated package.
- Do not ignore `public/`; it contains intentional UI assets.
