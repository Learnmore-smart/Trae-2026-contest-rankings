# .env

> Last updated: 2026-06-30 | Protection: CRITICAL

## Purpose

Stores local runtime environment variables for the TRAE contest ranking app.

## What It Does

- Provides server-side AI provider credentials, scraper settings, admin/cron shared secrets, and Firebase Admin credentials.
- Keeps non-secret runtime configuration visually separated from actual credentials and tokens.
- Preserves real secret values only in `.env`; this document must never contain secret values.

## Important Notes / NEVER Change

- Never commit `.env` or real secret values.
- Keep the file grouped under only `# public` and `# secrets` unless the user asks for more sections.
- Put model names, base URLs, retry/rate limits, scrape limits, project IDs, and app metadata under `# public`; these are configuration, not credentials.
- Keep NVIDIA text model order in `NVIDIA_PRIMARY_MODEL` + `NVIDIA_FALLBACK_MODELS`; keep image/multimodal preference in `NVIDIA_IMAGE_MODEL`.
- Current preferred NVIDIA text order is Kimi K2.6, GLM 5.1, then DeepSeek V4 Flash.
- Put only API keys, admin/cron tokens, and Firebase service-account material under `# secrets`.
- Do not expose API keys, Firebase service account JSON, admin tokens, or cron secrets to client code.

- 2026-07-04 Kiro: ROOT CAUSE of the "scoring stuck at 3973 / 开始评分 not working" — ALL AI providers are out of quota, not a code bug. Verified live: friend gateway key → HTTP 401 "Invalid token" (dead); BOTH NVIDIA keys → HTTP 429 "Too Many Requests" that does NOT clear after 65s (a daily/long-window quota cap, ~4000 topics × ~5 calls ≈ 20k calls burned today), not a per-minute rpm blip. With `AI_MAX_RATE_LIMIT_RETRIES=0` (unlimited) the judge retried the persistent 429 forever, so the pipeline never finished and the in-memory `running` lock in `app/api/trae-contest/run/route.ts` never released → the button became a permanent no-op. Fixes: (1) `AI_MAX_RATE_LIMIT_RETRIES` 0→8 so a persistent 429 falls through and the pipeline finishes / button unlocks instead of hanging; (2) `TRAE_JUDGE_CONCURRENCY` 12→4 to fit the 2-key (~80 rpm) capacity now that the high-limit friend gateway is gone. IMPORTANT: use `judge` (mode `unjudged`), NOT `judge:changed` — `changed` re-judges already-scored topics, and while quota is exhausted those re-judges fail and flip good topics to JUDGE_ERROR (totalScore -1), which is why the live "已评分" count dropped 3973→3940 during my `changed` run. No real fix is possible until quota returns (wait for NVIDIA reset, add NVIDIA_API_KEY_3.., or restore the friend key). No secret values were printed or changed.
